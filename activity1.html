Quick Legend

  • Client Apps: Interfaces that send queries (SQL/NoSQL) or API calls.
  • Query Processor: Parses SQL, creates execution plans, optimizes joins and scans.
  • Transaction Manager: Ensures ACID properties, handles locks/MVCC.
  • Storage Manager: Manages buffer pool, pages, and I/O to physical storage.
  • Physical Storage: Actual disks/SSDs or cloud block objects.

Client & API Layer

Web apps, mobile clients, BI tools and external services connect here. Communication uses SQL over a DB protocol, or REST/gRPC built on top of database APIs.

Query Processor & Optimizer

Responsible for turning SQL into an optimized execution plan: parsing, semantic analysis, cost-based optimization, and plan selection (indexes, join orders).

Transaction Manager & Concurrency

Enforces ACID semantics. Manages locks or MVCC snapshots, coordinates commit with the storage manager (two-phase commit in distributed systems).

Storage & Buffer Manager

Maintains an in-memory buffer pool, orders reads/writes, implements write-ahead logging (WAL) for durability, and performs checkpointing to flush pages to disk.

Physical Storage

Local disks, RAID arrays, SSDs, or cloud block storage. In distributed DBs this layer includes object stores (S3) or distributed file systems.

Supporting Components

Indexes speed lookups (B-tree, hash, LSM trees). System catalog stores schema and metadata. Replication, backup, security, caching, and monitoring sit alongside.
