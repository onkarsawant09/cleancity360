Database Architecture Components

Understand the layers and components of modern database systems

What is Database Architecture?

Database architecture describes the design, structure, and components of a database system. It defines how data is stored, organized, and accessed.

Key Objectives:

  • Data integrity and consistency
  • Efficient data retrieval
  • Scalability and performance
  • Security and access control
  • Reliability and fault tolerance

External Level

This level represents the user's view of the database. It describes the part of the database relevant to a particular user or application.

Components:

  • User Interfaces
  • Application Programs
  • Query Tools

Conceptual Level

This level describes the logical structure of the entire database. It defines what data is stored and the relationships among data.

Components:

  • Database Schema
  • Data Models
  • Constraints

Internal Level

This level describes the physical storage of the database. It details how data is stored on storage devices and accessed by the system.

Components:

  • Storage Structures
  • Indexing Methods
  • Data Compression
