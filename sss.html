Database Architecture Components

Understanding the layered structure of database systems and their core components

External Level

User Interfaces

Applications, forms, reports, and query tools that interact with the database.

APIs & Connectors

ODBC, JDBC, and other drivers that enable applications to communicate with the DBMS.

Conceptual Level

Data Model

The logical structure of the entire database including entities, relationships, and constraints.

Schema

The overall design of the database describing tables, views, indexes, and relationships.

Internal Level

Storage Structures

How data is physically stored on disk (files, pages, records, and data organization).

Indexing

Data structures like B-trees and hash indexes that enable efficient data retrieval.

DBMS Core

Query Processor

Interprets queries, creates execution plans, and optimizes query performance.

Transaction Manager

Ensures ACID properties (Atomicity, Consistency, Isolation, Durability) for database transactions.

Understanding Database Architecture Layers

The Three-Level Architecture

Database architecture is typically organized into three levels that separate the user's view of data from the physical storage:

  • External Level: How users perceive the data - includes user interfaces and application views
  • Conceptual Level: The community view of the database - describes what data is stored and relationships
  • Internal Level: How the data is physically stored - deals with storage structures and file organization

Key Benefits of Layered Architecture

This separation of concerns provides several important advantages:

  • Data Independence: Changes at one level don't affect other levels
  • Security: Users only see data they're authorized to access
  • Customization: Different views for different user groups
  • Maintainability: Easier to modify storage structures without affecting applications
